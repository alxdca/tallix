# Environment Configuration

The backend loads environment variables using `dotenv-cli` which checks:

1. `../.env` (project root) - **recommended**
2. `backend/.env` (current directory) - override if needed

## Quick Setup

```bash
# From project root
cp .env.example .env

# Edit with your local settings
vim .env
```

## Configuration

See the root [`.env.example`](../.env.example) for all available variables and documentation.

## Local vs Docker

- **Local development**: Use root `.env` file
- **Docker**: Environment variables are set in `docker-compose.yml`
